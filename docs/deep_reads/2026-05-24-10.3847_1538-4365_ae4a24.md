# Detection and Classification of Astronomical Sources with Astro-RetinaNet in Crowded Stellar Fields

**来源**: Astrophysical Journal Supplement Series  
**主题**: 天体统计  
**相关性**: 6/10

### 一、子领域定位

- **本文属于天文学的哪一支**：属于 **galactic / stellar astronomy**（恒星与近邻星系结构）与 **time-domain / survey astronomy**（巡天数据处理）的交叉地带。这个子领域的核心科学问题是：银河系及近邻星系的恒星密度分布、颜色-星等图（CMD）结构、以及星团动力学演化。目前成熟度极高——测光与星表提取是天文观测的工业化基础设施，但在极端拥挤星场（如球状星团核心、星系盘面）中，由于源混叠导致的测光提取至今仍是未解的硬伤，直接导致后续物理推断（如恒星质量函数、初始质量函数 IMF）系统性失真。
- **本文在这个子领域里的位置**：它针对的是巡天数据预处理流水线中最底层的“源提取与去混叠”环节。文章不讨论高层物理推断，而是试图用深度学习目标检测修复底层测光漏检这一数据缺陷，为后续统计推断提供更完整的星表。

### 二、关键术语扫盲

1. **Blending（源混叠）**：多个天体在望远镜二维投影上落在同一个像素区域内，像素亮度是它们的叠加。类比：多个人同时站在一个像素化的聚光灯下，你只看到一团总光斑。
2. **Crowded stellar field（拥挤星场）**：天体面密度极高的区域（如球状星团核心、星系盘面），每平方角分内可能有上万颗星，混叠不可避免。
3. **Completeness（完备度 / 源恢复率）**：给定真实存在的天体总数，测光算法能成功识别出多少比例。这是衡量选择偏差最直接的指标。
4. **Magnitude（星等）**：天体亮度的对数度量，数值越大越暗。差5个星等亮度差100倍。
5. **Photometry（测光）**：从图像像素中提取天体位置与亮度的过程，是天文数据处理的基石。
6. **Source extraction / detection（源提取）**：在图像中寻找天体信号并生成星表（catalogue）的算法步骤，传统主流工具是 SExtractor。
7. **Survey（巡天）**：对大片天区进行系统性成像观测的项目，产生海量图像数据。
8. **CSST（中国空间站望远镜）**：即将发射的空间光学望远镜，本文使用的模拟数据来源，相当于中国版哈勃+广角。
9. **HST / Hubble Space Telescope（哈勃空间望远镜）**：在地球大气干扰之上成像的空间望远镜，分辨率极高，本文用其真实观测的 M31 数据做验证。
10. **Focal Loss**：一种针对极端类别不平衡设计的损失函数，通过压低易分类样本的梯度贡献，强迫模型关注稀少类（本文中指混叠星）。
11. **Initial Mass Function (IMF)（初始质量函数）**：恒星形成时按质量分布的概率密度函数，是天体物理核心推断目标，极度依赖低质量暗星的完备度。
12. **PSF（Point Spread Function / 点扩展函数）**：一个理想点源在望远镜图像上实际形成的二维光斑轮廓，由光学衍射与大气抖动决定。

### 三、天文学家关心的问题

天文学家在拥挤星场中追问的核心是：**底层的恒星分布到底是什么？** 具体而言，他们想从观测图像推断出恒星的质量函数（尤其是低质量端的斜率）、星团的动力学状态、以及恒星演化的物理参数。这些推断的前提是拿到一份尽可能完备且无偏的星表。

当前领域的主流分析方法是基于阈值分割与轮廓拟合的传统源提取软件（如 SExtractor、DAOPHOT）。它们在稀疏场表现良好，但在拥挤场中面临已知局限：当星体间距小于 PSF 半径时，算法无法区分混叠源，导致大量暗星漏检（completeness 崩塌），或者把一个混叠团误认为单个亮源，造成星表系统性偏倚。这种底层漏检直接向上传导，使得基于星表的高层物理推断（如 IMF 测定）产生几十百分比的误差。天文学家极度渴望一种能从像素层面“解开”混叠团的方法，但目前多数尝试仍停留在对传统算法参数的微调或简单的 CNN 二分类，对多源混叠束手无策。

### 四、数据问题

- **数据来源**：CSST 模拟图像（模拟球状星团 NGC 2298）与 HST 真实观测图像（M31 核心区）。
- **数据形态**：单波段二维 imaging（灰度图像），像素矩阵。量级为几百到上千像素的局部星
